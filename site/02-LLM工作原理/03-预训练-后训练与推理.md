# 03 · 预训练、后训练与推理

面对事实错误或工具误用，人们常把问题笼统归因于“模型不够强”，然后尝试更大的模型、更高推理强度或更多 Prompt。Agent 工程需要更精确的判断：哪些能力来自权重，哪些信息只存在于本次 Context，哪些事实必须通过检索或工具从权威系统取得。

本章沿模型生命周期梳理这些来源，但不讨论训练流水线实现。目标是为选模型、组织 Context 和设计行动门禁建立边界，避免把统计上的行为倾向误写成系统保证。

## 学习目标

- 知道模型能力和行为倾向从哪里来。
- 区分权重学习、上下文学习、检索与工具执行。
- 理解推理时计算（test-time compute）是权衡轴，不是正确性证明。

## 1. 模型生命周期

一种常见但非唯一的流程：

1. **预训练**：在大规模语料上优化 Token 预测，获得语言、知识和模式能力。
2. **持续预训练或中期训练**：适配领域、长上下文、多模态或数据分布。
3. **监督微调（Supervised Fine-Tuning，SFT）**：从示范学习指令遵循和目标输出形式。
4. **偏好优化**：通过人类或 AI 偏好调整回答倾向，方法可包括人类反馈强化学习（Reinforcement Learning from Human Feedback，RLHF）、直接偏好优化（Direct Preference Optimization，DPO）等。
5. **推理强化**：利用可验证奖励或其他反馈强化某些解题策略。
6. **蒸馏**：把强模型行为迁移到较小模型。

闭源模型的具体数据和训练配方通常无法确认。工程结论应基于公开接口与测量，不把推测写成事实。

## 2. 四种能力来源

| 来源                                                    | 是否改权重 | 生命周期     | 适合什么      |
| ----------------------------------------------------- | ----: | -------- | --------- |
| 预训练/后训练                                               |     是 | 模型版本     | 通用能力与行为倾向 |
| 上下文示例（in-context examples）                            |     否 | 当前请求     | 临时说明任务与格式 |
| 检索增强生成（Retrieval-Augmented Generation，RAG）/ Knowledge |     否 | 外部数据生命周期 | 动态、可引用事实  |
| 工具（Tools）                                             |     否 | 执行时      | 查询和改变外部世界 |

频繁变化的价格、权限或订单状态不应依赖模型权重记忆。

## 3. 基础模型、指令模型与推理模型

- 基础模型（base model）更接近续写训练目标。
- 指令模型（instruction model）经过后训练，更倾向按角色和指令完成任务。
- 推理模型（reasoning model）可以在推理时投入更多计算，适合复杂规划、歧义和多步问题。

这些是行为差异，不是安全等级。更强的指令跟随仍不能替代授权，更高 reasoning effort 也不会证明最终动作正确。

## 4. 为什么模型会幻觉

模型的直接目标是产生符合条件分布的序列，不是查询一个带事务和来源约束的事实库。缺失证据、冲突上下文、训练模式和错误前缀都可能产生流畅但不受支持的结论。

系统缓解依赖：检索来源、要求可核验输出、允许 abstain、工具查询、确定性规则和评测，而不是寻找一句万能 Prompt。

## 5. 推理痕迹的边界

可见推理、reasoning summary 或模型解释不能默认视为完整忠实的内部因果轨迹。应用应审计：

- 输入与上下文版本。
- 结构化计划或状态。
- 工具提案、参数、策略结论与真实效果。
- 证据引用和简洁决策摘要。

不要把隐藏 Chain-of-Thought 当作产品日志或授权依据。

## 微实验

对同一组任务比较普通模型和 reasoning model：事实查询、含糊任务、多步规划、严格 Schema。记录质量、拒绝、延迟、Token、成本与 trial 方差。不要只看最好的一个输出。

## 常见误区

- 预训练是对互联网的可精确查询压缩。
- SFT 可以持续注入动态业务事实。
- In-context learning 会永久更新权重。
- 模型越大越真实、越安全。
- reasoning effort 越高对所有任务越好。

## 本章小结

权重学习提供通用能力与行为倾向，Context 提供临时任务信息，检索提供动态证据，工具负责查询或改变外部世界；这些层不能互相替代。下一章聚焦一次调用真正可用的信息边界，说明为什么标称长窗口、低温采样和自动摘要都不是可靠性保证。
