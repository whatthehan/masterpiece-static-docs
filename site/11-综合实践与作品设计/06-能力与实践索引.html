#06 · 能力与实践索引：把阅读转化为工程证据

读完相关章节，只代表接触过这些概念；工程理解还需要通过图、代码、Fixture、反例或故障结果来验证。本章把全书的三层主线映射到 Resolution Desk：LLM 底层解释模型能力与限制，Agent 层负责状态、工具、知识和行动，Agentic UI 层把权威状态变成人可以理解和控制的界面；Eval、安全、可靠性与可观测证据横切三层。迁移到其他产品时，再按真实能力和风险裁剪条目。

#1. 如何使用这份索引

Resolution Desk 主线先完成正常、拒绝和恢复三类纵向切片，再用三列信息检查理解是否落地：

  • 建议实践给出最小工件或实验；
  • 可观察结果说明机制成立时应该看到什么；
  • 常见失败信号帮助识别只会复述术语、却没有守住边界的实现。

标记“安全关键”的主题涉及权限、数据隔离或真实副作用，不能只用书面说明代替故障实验。Resolution Desk 始终使用 Mock、Recorded Fixture 或纸面时序，不连接真实支付与生产订单；与迁移后场景无关的主题可以删除，不必为了覆盖索引制造功能。

#5. 主线二与安全边界：Identity、Action 与 Protocol

主题建议实践可观察结果常见失败信号
身份与委派（安全关键）actor–tenant–resource–action–purpose 矩阵、带 Resource/Path/Origin/额度/期限约束的 Capability Snapshot、Delegation Chain身份由可信代码注入，Resource Service 再授权；未知 Tool Fail-closed；子任务所有约束维度只缩不增共享高权限 API Key；只比较 read/write/execute 动词；信任 authorized=true；把未知 Tool 当只读
Approval（安全关键）不可变 Proposal 与 Approval Record参数、目标、版本或期限变化后旧 Approval 失效“允许 Agent 使用工具”式泛化审批
Idempotency / Receipt（安全关键）Intent、Attempt、幂等记录与 ACK-loss 时序同一 Intent 多次投递只产生一次可观察效果Timeout 后换 Key Retry
MCP BoundaryHost/Client/Server 与信任边界图能说明 MCP 解决互操作，不替代 Runtime、Policy 或 Sandbox接入 MCP 就认为工具可信
Agent Skill（安全关键）approved publisher/version/digest、progressive disclosure、trigger/outcome eval加载内容与脚本可追踪，Skill 声明不能扩大 Tool 权限把 Markdown 指令或 allowed-tools 当作授权
Dynamic Tool Discovery（安全关键）discover→shortlist→authorize→load→execute→observe 与 Recall@k/Exposure EvalContext 缩小且 Unauthorized Exposure 为 0，无结果时安全停止搜到 Tool 就执行；把远端 Description 当可信指令
MCP ExtensionsApps sandbox/CSP/Host Proxy、Tasks Handle Fixture、Auth Extension Claims可展示不等于可信确认；Task 完成不等于业务 Outcome；Token 不替代资源授权iframe 当受信 UI；MCP Task 替代 Workflow；机器 Token 冒充用户
A2A Boundary（安全关键）AgentCard、Task Reducer、A2A v1 Wire Fixture、断线后 GetTask 对账，以及相对单 Agent Baseline 的 Admission PlanRemote Agent 的身份、Scope、预算、终态和 Artifact 都可追踪；不采用远程分支也能说明删除理由把 Remote Agent 当普通函数；SSE 断开后直接重建 Task；协议可用就默认上线
Threat Model（安全关键）数据流、资产、攻击者、滥用故事和不变量每条不变量都有确定的执行位置与真实 Outcome Eval只列 OWASP 名称
Prompt Injection（安全关键）Direct/Indirect/Persistent/Cross-agent 攻击链每条至少两个独立阻断，模型失效后仍无越权效果只过滤固定字符串
Agent Red Team（安全关键）Threat Hypothesis、攻击 Harness、确定性 Outcome、最小复现与 Regression 入库核心 Tool/MCP/Memory/AG-UI/A2UI/Native UX 攻击可测；生产启用 Multi-Agent/A2A 时追加远程攻击链；模型识别与系统阻断分别评分只做聊天越狱；攻击成功后没有回归样本和 owner
Sandbox / Sink（安全关键）File/Network/Process/Secret/Resource 控制矩阵、Canonical Resource、requested/effective policy 对拍SSRF、Path、symlink、Shell、旁路子进程、egress 和资源耗尽测试通过；授权、锁与 I/O 使用同一资源身份把配置声明当 Enforcement；把 Container 或 Rust 当安全边界
Incident ResponseDetect–Contain–Establish Facts–Recover–Learn PlaybookKill Switch 后在途效果仍被核对，已确认的事件模式回流 Eval所有 Run 被批量写成 Failed

#7. 横切证据：Reliability、Observability 与 Operations

主题建议实践可观察结果常见失败信号
Timeout / Retry / Cancel（安全关键）Execution/Effect 双轴、Retry 放大计算、Reconciliation 时序Cancel 不等于 Rollback；Command Timeout 不盲试Timeout 直接写失败未执行
BackpressureFan-out 上界、Bounded Queue、Bulkhead、Transport/Memory/Artifact/Context/UI 多层输出预算与满队列语义任一输入都有并发、排队、Token、费用和输出保留上界；并发完成顺序不改变稳定 Join 的 Canonical ResultPromise.all 当并发控制；只在进入 Context 前截断

Exactly-Once 边界Delivery/Handler/Commit/Effect 四层说明多次 Attempt 只对应一个可观察的业务效果宣称 Queue 的 Exactly-Once 等于第三方效果一次
Trace / Audit / Eval同一 run_id 的因果链和责任链能从 Outcome 追到 Context、Approval、Tool 与版本日志很多但无法复盘
OpenTelemetry PropagationHTTP→Queue→Worker→Model/MCP 的 Trace Context、低基数 Span 名称与脱敏 AttributeCritical Path 与累计 fan-out 成本可见，Trace/Audit/Eval 各守边界只在 Web Handler 建 span；Prompt 和 Token 泄露到普通 Trace
SLO / CostTask Success、Time to Truth、P95/P99、Cost per Successful Task安全零容忍不被普通错误预算稀释只看 Model Uptime 和 Token 单价
Model Routing / CascadeRoute Policy、Risk/Capability/Region 约束、固定模型对照与 Router EvalProtected Slice 无错误下放，升级率与单位成功任务成本可解释模型自报难度决定权限；只比较请求均价
ReleaseBehavior Bundle、Offline/Shadow/Canary/Rollback 记录Shadow 无副作用；旧 Run 可恢复；Rollback 恢复完整行为只回滚容器镜像
Controlled Improvement（安全关键）Signal/Finding/Eval Target/Change Proposal/Candidate Bundle 与权限矩阵Optimizer 不见 Holdout、不能改 Grader/Policy/发布；失败候选不上线，退化版本可回滚把生产反馈、Reflection 或自动 Prompt 结果直接写入在线配置
Provider FailureFallback 或非 Fallback 故障策略采用 Fallback 时通过同一 Eval；不采用时有界等待/拒绝/人工可演练故障时临时换模型
Drain / Disaster RecoveryWorker Drain、RTO/RPO 与恢复演练未知效果不丢失，Queue Replay 不重复副作用关闭连接就算安全停机
Production TopologyApplication Server/Worker/Queue/Store/Tool Gateway 责任图、Probe、Fencing 与迁移矩阵SSE 可重连、Worker 可接管、旧 Event/Checkpoint 可恢复或转人工Pod Running 被当成业务健康；Sticky Session 替代持久状态

#9. 两种使用方式

按主线构建 Resolution Desk 时，这份索引用于检查每个里程碑是否同时留下概念、运行与故障证据。完整节奏见八周构建路径，最终验收条件见总装与验收。

把能力迁移到其他场景时，根据真实输入、Action 和 Outcome 裁剪索引；安全关键项不能删除，场景无关的协议和高级能力不必保留。具体方法见从 Resolution Desk 迁移到其他 Agent 场景。
