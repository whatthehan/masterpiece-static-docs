#06 · 能力与实践索引：把阅读转化为工程证据

读完相关章节，只代表接触过这些概念；工程理解还需要通过图、代码、Fixture、反例或故障结果来验证。本章把全书知识映射到 Resolution Desk 的验收证据；迁移到其他产品时，再按真实能力和风险裁剪条目。

#1. 如何使用这份索引

Resolution Desk 主线先完成正常、拒绝和恢复三类纵向切片，再用三列信息检查理解是否落地：

  • 建议实践给出最小工件或实验；
  • 可观察结果说明机制成立时应该看到什么；
  • 常见失败信号帮助识别只会复述术语、却没有守住边界的实现。

标记“安全关键”的主题涉及权限、数据隔离或真实副作用，不能只用书面说明代替故障实验。Resolution Desk 始终使用 Mock、Recorded Fixture 或纸面时序，不连接真实支付与生产订单；与迁移后场景无关的主题可以删除，不必为了覆盖索引制造功能。

#2. 任务、模型与实验基础

任务契约与 Baseline用户、输入、Outcome、禁止动作、预算；30–50 个 Case；非 Agent BaselineCase 可重复，Outcome 不依赖模型自评，Dataset 有版本只有 Demo Prompt；成功标准是“看起来不错”
概率与 Sampling同一 Case 的多 Trial 和分布能区分高 Token Probability、低随机性与事实正确只跑一次就比较模型
Token 与 ContextTokenization 记录、截断与位置干扰实验能解释容量、有效利用和成本差异把 Context Window 当无限可靠 Memory
Transformer / KV CachePrefill / Decode / Cache 图不把 Causal Mask 当正确性，不把 KV Cache 当长期状态用模型原理替代应用验证
Outcome / Trajectory Eval分层 Grader 与 Trace 示例权威业务结果和执行轨迹分别评分只判断最终文本；LLM Judge 未校准
实验设计Dataset 分区、多 Trial、区间、版本与停止规则不把相关 Trial 当独立样本，不污染 Holdout选择性报告最好结果

Synthetic Case / Human Review来源 Manifest、约束生成、去重、污染检查与分层人工抽样合成样本补足稀有 Slice，人类分歧和 rubric 缺陷可见把生成量当覆盖率；用同一模型生成并评分

#4. Context、Knowledge 与 Memory

主题建议实践可观察结果常见失败信号
Context Builder可执行 Builder 与 Context Manifest每项有 Source、Trust、Freshness、Budget 和纳入理由把全部历史拼进 Prompt
Compaction / Reset / Isolation前后对拍 Fixture 与丢失检查关键约束、未决事项和证据引用不丢失摘要覆盖原始事实
Provenance 与 FreshnessSource Schema、冲突与过期处理表结果能追到版本化来源，冲突不会由模型静默决定只有相似度分数
ACL-scoped Retrieval（安全关键）检索前 Tenant/ACL 图、Recall/Precision Eval无权内容从候选生成前排除，后置检查只做兜底Top-k 后才过滤 ACL
Memory Policy（安全关键）Candidate/Write/Read/Update/Delete 规则来源、置信、TTL、冲突与删除传播完整模型反思直接写长期事实
删除验证（安全关键）Thread、Chunk、Vector、Cache、Trace、Eval 与 Backup 映射Canary ID 在所有可服务副本中不可见只删除聊天记录

#5. Identity、Action 与 Security

主题建议实践可观察结果常见失败信号
身份与委派（安全关键）actor–tenant–resource–action–purpose 矩阵、Delegation Chain身份由可信代码注入，Resource Service 再授权共享高权限 API Key；信任 authorized=true
Approval（安全关键）不可变 Proposal 与 Approval Record参数、目标、版本或期限变化后旧 Approval 失效“允许 Agent 使用工具”式泛化审批
Idempotency / Receipt（安全关键）Intent、Attempt、幂等记录与 ACK-loss 时序同一 Intent 多次投递只产生一次可观察效果Timeout 后换 Key Retry
MCP BoundaryHost/Client/Server 与信任边界图能说明 MCP 解决互操作，不替代 Runtime、Policy 或 Sandbox接入 MCP 就认为工具可信
Agent Skill（安全关键）approved publisher/version/digest、progressive disclosure、trigger/outcome eval加载内容与脚本可追踪，Skill 声明不能扩大 Tool 权限把 Markdown 指令或 allowed-tools 当作授权
Dynamic Tool Discovery（安全关键）discover→shortlist→authorize→load→execute→observe 与 Recall@k/Exposure EvalContext 缩小且 Unauthorized Exposure 为 0，无结果时安全停止搜到 Tool 就执行；把远端 Description 当可信指令
MCP ExtensionsApps sandbox/CSP/Host Proxy、Tasks Handle Fixture、Auth Extension Claims可展示不等于可信确认；Task 完成不等于业务 Outcome；Token 不替代资源授权iframe 当受信 UI；MCP Task 替代 Workflow；机器 Token 冒充用户
A2A Boundary（安全关键）AgentCard、Task Reducer、A2A v1 Wire Fixture、断线后 GetTask 对账Remote Agent 的身份、Scope、预算、终态和 Artifact 都可追踪把 Remote Agent 当普通函数；SSE 断开后直接重建 Task
Threat Model（安全关键）数据流、资产、攻击者、滥用故事和不变量每条不变量有 enforcement point 与真实 Outcome Eval只列 OWASP 名称
Prompt Injection（安全关键）Direct/Indirect/Persistent/Cross-agent 攻击链每条至少两个独立阻断，模型失效后仍无越权效果只过滤固定字符串
Agent Red Team（安全关键）Threat Hypothesis、攻击 Harness、确定性 Outcome、最小复现与 Regression 入库核心 Tool/MCP/Memory/Native UX 攻击可测；启用 Multi-Agent/A2UI 时追加对应攻击链；模型识别与系统阻断分别评分只做聊天越狱；攻击成功后没有回归样本和 owner
Sandbox / Sink（安全关键）File/Network/Process/Secret/Resource 控制矩阵SSRF、Path、Shell、egress 和资源耗尽测试通过把 Container 或 Rust 当安全边界
Incident ResponseDetect–Contain–Establish Facts–Recover–Learn PlaybookKill Switch 后在途效果仍被核对，真实 Case 回流 Eval所有 Run 被批量写成 Failed

#6. Reliability、UX 与 Operations

主题建议实践可观察结果常见失败信号
Timeout / Retry / Cancel（安全关键）Execution/Effect 双轴、Retry 放大计算、Reconciliation 时序Cancel 不等于 Rollback；Command Timeout 不盲试Timeout 直接写失败未执行
BackpressureFan-out 上界、Bounded Queue、Bulkhead 与满队列语义任一输入都有并发、排队、Token 和费用上界Promise.all 当并发控制
Durable ExecutionCheckpoint、Event History、Activity、双 Worker 时序新 Epoch 接管，旧写入被 Fencing/CAS 拒绝Checkpoint 只是最后一条消息
Exactly-Once 边界Delivery/Handler/Commit/Effect 四层说明多次 Attempt 收敛为一个业务效果宣称 Queue 的 Exactly-Once 等于第三方效果一次
Agent UXPublic Run State、六个恢复界面、Control EventUI 不发明终态；Stop/Retry/Resume 与 Runtime 一致Loading/Error 两个状态承载所有任务
A2UI Renderer（启用时安全关键）固定 Catalog、Envelope/Graph Validator、Surface Reducer、Action Gateway未知组件、危险 URL、伪造 Action 与超量 Payload 均被拒绝执行模型生成的 JSX；动态加载 Catalog；信任客户端 Authorization
Trace / Audit / Eval同一 run_id 的因果链和责任链能从 Outcome 追到 Context、Approval、Tool 与版本日志很多但无法复盘
OpenTelemetry PropagationHTTP→Queue→Worker→Model/MCP 的 Trace Context、低基数 Span 名称与脱敏 AttributeCritical Path 与累计 fan-out 成本可见，Trace/Audit/Eval 各守边界只在 Web Handler 建 span；Prompt 和 Token 泄露到普通 Trace
SLO / CostTask Success、Time to Truth、P95/P99、Cost per Successful Task安全零容忍不被普通错误预算稀释只看 Model Uptime 和 Token 单价
Model Routing / Cascaderoute policy、risk/capability/region gate、固定模型对照与 router evalProtected Slice 无错误下放，升级率与单位成功任务成本可解释模型自报难度决定权限；只比较请求均价
ReleaseBehavior Bundle、Offline/Shadow/Canary/Rollback 记录Shadow 无副作用；旧 Run 可恢复；Rollback 恢复完整行为只回滚容器镜像
Provider FailureFallback 或非 Fallback 故障策略采用 Fallback 时通过同一 Eval；不采用时有界等待/拒绝/人工可演练故障时临时换模型
Drain / Disaster RecoveryWorker Drain、RTO/RPO 与恢复演练未知效果不丢失，Queue Replay 不重复副作用关闭连接就算安全停机
Production TopologyApplication Server/Worker/Queue/Store/Tool Gateway 责任图、Probe、Fencing 与迁移矩阵SSE 可重连、Worker 可接管、旧 Event/Checkpoint 可恢复或转人工Pod Running 被当成业务健康；Sticky Session 替代持久状态

#7. Rust 迁移与场景专项

主题建议实践可观察结果常见失败信号
Rust 迁移验收Profile、SLO、候选边界、成功阈值与停止条件问题确实位于本地组件，低成本替代方案已比较理由只是“Rust 更快”
Cross-language Contract（安全关键）Wire Type、Error、Identity、Golden / Property Test金额、时间、Null、Unicode、Cancel 与 Unknown Effect 两侧一致共享 JSON 就认为语义相同
Migration ReleaseRecord/Replay、Shadow、Canary、RollbackOutcome、Trace、Audit、Latency 与资源均达到预注册标准只比较返回文本
Specialty Admission用户价值、简单 Baseline、新状态/风险、Eval 与删除条件复杂度相对 Baseline 有可测收益因流行引入 Multi-Agent / Memory / Computer Use

#8. 两种使用方式

按主线构建 Resolution Desk 时，这份索引用于检查每个里程碑是否同时留下概念、运行与故障证据。完整节奏见八周构建路径，最终门禁见总装与验收。

把能力迁移到其他场景时，根据真实输入、Action 和 Outcome 裁剪索引；安全关键项不能删除，场景无关的协议和高级能力不必保留。具体方法见从 Resolution Desk 迁移到其他 Agent 场景。

#本章小结

这份索引的价值是把“理解过”转换成“能够重建、实现和证伪”。可以先使用综合能力自测定位知识缺口，再按项目风险选择实验；截图和 Demo 不能替代真实 Outcome 与故障恢复。
