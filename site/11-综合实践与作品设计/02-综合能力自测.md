# 02 · 综合能力自测：能否独立解释一个 Agent 系统

这份自测用于诊断知识结构，不是术语记忆竞赛，也不是生产认证。完成第一个 TypeScript Agent Runtime 后，可以先不查资料作答，再使用下一章的判断要点核对理解。

不必一次完成全部题目。概念题适合检查术语边界，故障题适合团队设计评审，系统题可以直接转化为实现任务。答案应同时说明“机制保证什么”和“机制不保证什么”；场景题需要写出状态转移、执行约束和真实 Outcome，不能只给一段 Prompt。

## 使用规则

- 第一遍尽量使用空白文档作答，再按需回到对应章节；
- 所有架构图标出权威状态、信任边界和副作用执行点；
- 遇到效果未知时明确标记 `unknown`，不使用模糊措辞掩盖；
- 核对后保留原始答案，将错误或遗漏整理成“错误模型清单”；
- 能够完整回答这些问题，只能说明系统心智模型已经基本连贯，不能替代真实的恢复、发布和安全证据。

## A · 核心概念（20 题）

5. 标称 Context Window 为什么不等于任务的有效 Context？
6. 后训练与较低 Sampling Temperature 可能改善什么，不能承诺什么？
7. 为什么不应把原始 Chain-of-Thought 当作审计证据？应记录哪些可验证信息？
8. 固定 Workflow、带工具的 LLM 调用与 Agent Loop 在控制权上有何区别？
9. 任务契约、版本化案例集与非 Agent Baseline 分别解决什么问题？
10. JSON Schema / Structured Outputs 能保证什么？为什么 Missing、`null`、Default 与 Delete 必须分开？

12. Streaming API 中，为什么 TCP Chunk 不等于语义 Event？为什么只能执行完整 Tool Item？

## B · 故障场景（6 题）

### 21. ACK 丢失与 Cancel 竞态

退款服务已经 Commit，ACK 在返回途中丢失，Runtime Timeout；用户紧接着点击 Stop。写出安全状态转移、核对顺序、用户文案和明确禁止的动作。

### 22. Indirect Prompt Injection 与 SSRF Sink

研究 Agent 在网页中读到恶意指令，随后生成了 Schema 合法的 URL，目标指向内网或云 Metadata Endpoint，并试图携带 CRM 数据。设计 Model、Policy 和 Execution Environment 三层防御，以及 Outcome / Trajectory Eval。

### 23. Approval TOCTOU

用户批准订单 A 退款 10 元，模型随后把目标改成订单 B、金额改成 1000 元。哪些字段、状态和服务端检查必须让执行失败？

### 24. 级联 Retry

API Gateway、Agent Runtime 和 Tool Client 都配置 `max_attempts = 4`。最底层最多收到多少个 Attempt？若三层配置的是“首次之外最多 Retry 4 次”，答案又是多少？应如何重新设计？

### 25. ACL Retrieval 与 Memory 删除

Vector Search 先从全租户选 Top-k，再过滤 ACL，结果被其他租户的高相似内容占满；同时，用户已删除的偏好仍从 Cache 或 Vector Index 命中。分别说明泄露风险、Recall Starvation、删除传播和验证方案。

### 26. Stream 断连与 Backpressure

Tool Call 的 Arguments 只到达一半时 Stream 断开；重连后服务器重放部分 Event，而下游队列已经满。设计 Event Reducer、去重/缺口处理、完整性门禁、Backpressure、Deadline/Cancel 与 UI 状态。

## C · 系统设计（3 题）

### 27. 有界 Agent Loop

画出包含 Model、Tool、补充输入、Approval、Cancel、Unknown Effect、Reconciliation、Budget 和终态的状态机。给出至少 10 行 `state × event → guard / effect / next` 转移表，并列出八条可测试不变量。

### 28. Eval 与实验方案

为“研究政策—判断资格—生成退款提案—审批后执行”的任务设计首批 30–50 个 Case。方案应包含任务契约、非 Agent Baseline、Case Slice、真实 Outcome / Trajectory Grader、人工金标与 Judge 校准、多 Trial、Dataset 分区与版本、区间估计、成本和发布门禁。

### 29. TypeScript + Node Runtime

设计一个不使用 Agent Framework 的 Runtime。说明 Discriminated Union、Thread/Run/Item/Event 存储、Stream Reducer、状态转移、`AbortSignal` 树、Semaphore / Bounded Queue、Deadline / Retry Budget、Tool Adapter、Trace / Eval Hook 和测试方案；同时说明 CPU 密集或阻塞任务如何隔离。

## D · 协议专项（4 题，选做）

### 30. SSE、OpenAI Streaming 与 AG-UI

说明 HTTP Chunk、SSE Frame、OpenAI Responses Event、Canonical RunEvent 和 AG-UI Event 的层次。一次 Tool Loop 包含两次模型 Response 时，哪些完成事件分别能闭合 Text Item、Provider Response 和外层 Agent Run？

### 31. A2UI Renderer

设计一个只能使用本地 Design System 的 A2UI v0.9.1 Renderer。说明 Surface、Catalog、Component Graph、Data Model 与 Action 的权威边界，并列出至少六种恶意或故障 Fixture。

### 32. MCP 与 A2A

为什么“远端安全审计 Agent”不一定适合包装成一个普通 MCP Tool？使用 A2A 时，说明 AgentCard Discovery、Message / Task 选择、Interrupted / Terminal State、Artifact、断线恢复和 Authorization 的处理。

### 33. 开放协议的版本漂移

A2A v1.0 Normative Proto、A2UI Extension 示例与 A2A JavaScript SDK 支持版本不一致时，应以什么作为 Wire Contract？如何使用 Adapter、Capability / Version Negotiation、Golden Fixture 和发布门禁避免静默混用？

## 结果如何使用

自测的价值在于暴露错误模型，而不是计算总分。以下错误会直接破坏系统边界，应优先回到对应章节和故障实验修正：

- 把 Schema 合法当成业务正确、URL 安全或执行授权；
- 在 Timeout 或 Cancel 后假定副作用没有发生，并使用新幂等键盲目 Retry；
- 先全局检索 Top-k，再过滤当前用户无权读取的内容；
- 允许模型直接执行 Tool，或允许终态继续产生动作；
- 执行不完整的 Stream Item，或让并发、Deadline 和费用没有上界；
- 只评最终文本，不保留非 Agent Baseline、真实 Outcome 和多 Trial 证据。

概念遗漏可以通过最小实验补齐；涉及授权、跨租户数据、未知效果和恢复语义的错误，则应先修正实现与测试，再继续扩大工具权限或生产流量。
