#04 · 状态、记忆与压缩

检索管线负责从外部知识中取得证据，长任务本身还会不断产生事件、计划、摘要与用户偏好。如果这些对象都被塞进聊天历史或向量库，订单真实状态可能被旧摘要覆盖，一次模型误判也可能被固化成跨会话“事实”。

本章为这条主线收口：事件日志和 Runtime State 持有已经发生与当前进行的事实，Context 只是一次调用的投影，长期记忆（Long-term Memory）则是经过写入、读取与删除门禁的派生资产。分清这些对象后，压缩才能缩短输入而不破坏恢复、审计与权限边界。

#学习目标

  • 区分事件日志、运行状态、Context、长期记忆和知识库。
  • 能设计长期记忆写入门禁。
  • 理解压缩是有损派生，不是权威状态。

#1. 五类对象

对象作用域内容权威性
Event LogRun/Thread调用、审批、执行回执、状态转移已发生事件的权威记录
Runtime StateRun/Thread当前阶段、预算、计划、待审批动作当前运行权威状态
Context一次模型调用精选输入临时、非权威
Long-term Memory跨 Thread偏好、稳定事实、经验取决于来源和治理
Knowledge Corpus文档/业务域有版本的外部事实由来源系统决定

#2. 实用记忆分类

  • Working：当前任务短期需要。
  • Semantic：较稳定事实或明确偏好。
  • Episodic：过去发生的事件与结果。
  • Procedural：规则、技能或操作方式。

这是设计分类，不要求为每类建一个数据库。

#3. 写入门禁

模型可以提出“建议记住”，但不能自动写入。至少判断：

  • 未来价值和适用范围。
  • 用户明确陈述还是模型推断。
  • 来源与可回溯证据。
  • namespace：用户、租户、项目、应用。
  • 敏感度、同意与保留期。
  • 稳定事实还是短期状态。
  • 冲突、置信度、有效期和 TTL。
  • 用户能否查看、更正和删除。

建议记录：

id, namespace, kind, structured_value,
source_refs, confidence, sensitivity, consent,
valid_from, valid_until, version, conflict_status

#4. 读取门禁

召回一条记忆不代表应该放进 Context。读取时重新检查 actor、purpose、scope、时效、冲突和当前任务相关性。

#5. 压缩与 Compaction

Compaction 应保留：

  • 当前目标与禁止项。
  • 已完成动作及不可变回执。
  • 待处理事项和失败原因。
  • 权限/审批范围与有效期。
  • 证据和原始工件引用。
  • 预算、版本和恢复位置。

摘要需标记为 derived artifact，并可回溯原始来源。模型生成的总结不能覆盖数据库中的权威状态。

#6. Memory 必须独立评测

  • Write precision：被写入的内容中有多少确实值得长期保存。
  • Retrieval precision/recall：相关合法记忆是否命中，无关记忆是否污染 Context。
  • Stale/conflict handling：过期、冲突和低置信记录是否被正确降级或拒绝。
  • Cross-scope leakage：跨用户、租户或项目违规率。
  • Downstream utility：加入记忆后真实任务是否优于无记忆 baseline。
  • Deletion effectiveness：删除后 memory、cache、index、summary 和 backup 读取路径是否都不再返回。

写入数量多不代表记忆好；错误持久化会把一次失败升级为跨会话系统性失败。

#前置桌面推演（45 分钟）

创建十条记忆：冲突、过期、推断、跨租户、用户已删除、敏感、无来源。设计写入和读取策略，确保只有合法、相关、未过期记录进入指定用户的 Context。

通过证据：为每条记录给出 write decision、read decision、理由和 grader；删除案例必须覆盖 cache/index/summary，而不只查主表。

#常见误区

  • 保存全部聊天就是长期记忆。
  • Vector DB 就是 Memory。
  • 模型最适合决定保存和删除什么。
  • 摘要可以覆盖原始执行回执。
  • 一条偏好一经保存就永久有效。

#本章小结

状态、Context、知识与长期记忆具有不同的权威性和生命周期；Compaction 只能生成可追溯的有损派生物，不能覆盖事件、回执或领域事实。完成信息面的治理后，下一阶段从工具契约与错误模型进入行动面，把模型建议接入受权限、幂等与故障语义约束的执行系统。
