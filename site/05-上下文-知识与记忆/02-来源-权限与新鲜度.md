# 02 · 来源、权限与新鲜度

## 学习目标

- 把 provenance、ACL 和 freshness 作为知识对象的一等字段。
- 能处理冲突证据、派生内容和删除传播。
- 理解“检索到了”不代表“允许使用”。

## 1. 知识对象的最小元数据

```text
resource_id
tenant / owner
source_system + source_uri
content_hash + version
created_at / observed_at / valid_from / valid_until
classification / sensitivity
acl / policy_tags
derivation / parent_refs
```

没有来源、版本和适用时间的知识无法可靠引用或失效。

## 2. Provenance 链

```text
source document
→ extracted chunk
→ normalized fact
→ retrieved evidence
→ generated claim
```

每层应能回溯上层。摘要、embedding 和生成答案都是派生物，不应丢掉 parent reference。

## 3. 权限传播

对文档的访问许可不自动等于：

- 可以把内容发送给第三方模型或工具。
- 可以跨租户写入共享记忆。
- 可以把个人数据用于 Eval。
- 可以长期保存摘要或 embedding。
- 可以让另一个 Agent 读取。

必须考虑 actor、purpose、destination、retention 和 transformation。

## 4. 新鲜度与冲突

不同数据需要不同更新语义：

- 价格、库存、权限：执行前即时查询。
- 政策文档：版本与生效区间。
- 用户偏好：允许修改、冲突与遗忘。
- 研究资料：记录获取日期和来源质量。

遇到冲突时不要让模型凭措辞“选一个更像真的”；按来源权威、版本、时间和业务规则解析，无法解析则展示不确定性或升级。

## 5. 删除与更正传播

删除请求可能涉及：原始库、chunk、index、embedding、memory、cache、trace、eval dataset、backup 和第三方。系统需要数据地图和 tombstone/重建策略，不能只删 UI 中的一条聊天记录。

## 微实验

让同一政策存在旧版、新版、草案版和另一租户定制版。设计检索过滤和引用规则；再模拟新版撤回与用户删除，列出所有必须更新的派生物。

## 常见误区

- 内部文档默认可以用于任何 Agent 目的。
- Embedding 不可逆，所以不算敏感数据。
- 最新写入时间就等于业务生效时间。
- 引用域名可信就无需验证具体版本。
- 删除原文后向量索引自然失效。
