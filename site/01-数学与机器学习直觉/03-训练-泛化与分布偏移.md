# 03 · 训练、泛化与分布偏移

## 学习目标

- 理解训练目标、产品目标和真实任务成功之间的间隙。
- 区分预训练知识、后训练行为与应用时上下文。
- 能识别数据泄漏、过拟合和生产分布偏移。

## 1. 经验风险不等于真实世界风险

机器学习从有限数据优化代理目标。训练损失下降，只说明模型更符合该数据和目标；产品关心的可能是事实正确、合规、任务完成和不产生错误副作用。

```text
训练数据分布 ──优化代理目标──> 模型
生产任务分布 ──业务与安全指标──> 系统结果
```

两个分布和两套目标都可能不一致。

## 2. 三种常见偏移

- Covariate shift：输入分布 `P(X)` 变化，如用户从短问题变成长文档。
- Label/prior shift：结果类别先验 `P(Y)` 变化，而类条件输入近似稳定。
- Concept drift：`P(Y|X)` 或判定规则变化，如政策更新后同一事实对应不同结论。
- Interaction/non-stationarity：Agent 的动作改变后续环境，使数据不再是静态独立样本。

Agent 的交互式非平稳尤其重要：模型的第一次错误动作会制造下一步的新输入。

## 3. 后训练改变倾向，不提供硬保证

SFT、偏好优化和强化学习可以强化指令跟随、风格、安全倾向或推理行为，但仍是统计性行为塑造。它们不能代替：

- 服务器端授权。
- 数据库约束与事务。
- 工具参数语义校验。
- 预算、超时和取消。
- 针对当前任务的评测。

## 4. 评测污染与过拟合

如果不断根据固定评测集改 Prompt，最终可能只适应该集合。应区分：

- Development set：用于定位和迭代。
- Regression set：覆盖已知故障。
- Holdout set：只在阶段门禁运行。
- Production shadow data：监控真实分布，但需隐私与权限治理。

模型或 Agent 可能利用 grader 漏洞“得高分但没完成真实目标”，因此必须检查环境最终状态。

## 微实验

为同一意图设计四种输入：标准、口语、信息缺失、恶意夹带。比较单一 Prompt 在四组上的表现。然后只针对第一组优化 Prompt，再重跑全部组，观察是否发生局部提升和整体回退。

## 常见误区

- 更大的通用模型自动更适合所有垂直任务。
- 后训练过的安全模型可以直接拿到生产写权限。
- 测试集得分高等于线上稳定。
- 收集更多案例总比修复任务定义好；含糊标签只会扩大噪声。
