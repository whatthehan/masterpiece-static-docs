#08 · 场景专项与全书毕业

八周结束时，你已经能运行并解释一个 L1 Workbench；全书结束时，你还要证明它能跨时间恢复、在 Provider 故障中退化、通过真实 UI 被人控制，并在一个具体场景中产生超过简单方案的价值。两者不是同一张毕业证。

L6 也不要求完成九个项目。它要求从九类能力中选择 1–2 个，用统一准入契约证明复杂度值得。

#1. 八周启动与全书毕业的区别

阶段已经证明尚未证明
八周启动正式 M0、L0 Adapter、L1 Runtime；Context、知识、受控行动薄切片生产恢复、Provider 切换、真实 UI、发布运营与场景专长
全书毕业L2–L5 生产证据 + 1–2 个 L6 专项收益不代表掌握所有场景或所有框架

八周闭卷检查可以暴露心智模型缺口，但全书毕业只认运行、故障、发布与用户结果证据。

#4. 全书毕业门禁

#任务与评测

  • 至少 100 个版本化案例，覆盖正常、模糊、拒绝、攻击、Provider/Tool 故障、取消、恢复和发布切流。
  • Outcome 与 Trajectory 分开评分；Protected Slice、安全不变量、尾延迟和每成功任务成本有明确门禁。
  • 多 Trial、Holdout、Judge 校准和生产样本治理均可复查。

#系统与用户控制

  • 可运行的 TypeScript/Node Agent Application，具备 Thread/Run/Item、Context Manifest、受控工具、权限/审批、预算和 Trace。
  • 真实 UI 展示等待输入、等待审批、执行、停止请求、未知效果、核对、部分完成与人工接管；按钮与持久事件一致。
  • 一个 Run 能等待跨小时事件并在 Worker 重启后恢复；幂等、Fencing/CAS 和权威查询证明没有重复副作用。

#Provider 与生产运营

  • Provider 故障策略经过故障注入：采用 Fallback 时，主/备 Model/Provider 通过同一 Eval 与协议 Contract，且不改写已审批提案；不采用时，有界等待、拒绝、有限结果或人工接管与声明一致。
  • Quota、Circuit Breaker、Backpressure、Stream Drain 和旧/新 Run 路由经过故障注入。
  • 发布状态机完成 Offline Eval、Shadow、Canary、Ramp、Rollback 与 Kill Switch 演练。
  • 灾备恢复满足声明的 RTO/RPO，配置、Prompt、Policy 与模型路由都可版本化回滚。

#场景专项与作品证据

  • 只需选择 1–2 个 L6 专项，但必须满足上表的输入、工具、状态、风险与 Eval 准入。
  • 作品仓库包含可运行说明、架构/信任边界、版本化 Dataset、故障矩阵、发布记录、关键取舍和明确不做项。
  • 外部评审者能够复现至少一条正常路径和一条失败恢复路径，而不是只观看 Demo。

Rust 迁移不是所有作品的毕业前提；一旦真实迁移，则另行通过 Rust 真实迁移门禁。

#5. 最后一条端到端验收

重新运行小林的场景：

order_123 / CNY 100.00
→ 两份政策冲突
→ 澄清并冻结提案
→ 精确审批
→ Commit 成功、ACK 丢失
→ 用户 Cancel
→ Provider 限流、Worker 重启
→ 新 Epoch 使用原幂等键 Reconcile
→ UI 显示真实结果
→ Trace/Audit/Eval/Cost 完成复盘
→ 修复版本经 Shadow/Canary/Rollback 演练

这条路径只有一个真实 Outcome，却同时验证 Context、Loop、Harness、授权、UX、可靠性与运营。如果其中任何一步只能用“模型应该会处理”解释，就还没有毕业。

#尾声：重新看见你熟悉的 Agent

回到全书开头：你让 Claude Code 或 Codex 修复一个失败测试。现在你看到的不再只是“模型读了文件然后写代码”。你能指出项目规则怎样进入 Context，搜索与测试怎样形成反馈 Loop，Permission 与 Sandbox 怎样约束行动，Diff 与 Test 为什么比完成声明更可信，也能判断何时值得加入 Reviewer、Outer Loop 或新的 Harness 组件。

面对小林的退款，你也不再问“模型够不够聪明”，而会继续追问：它看见了什么，谁允许它行动，状态在哪里持有，失败怎样收敛，结果由什么证据确认，新版本又凭什么进入流量。

这就是全书真正的毕业：不是背完一套术语，也不是复刻某个产品，而是能够把一个看似自然的 Agent 行为拆成可实现、可证伪、可运营的工程系统，并且始终知道哪些部分仍然未知。
