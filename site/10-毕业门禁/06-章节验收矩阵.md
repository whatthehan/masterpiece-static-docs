# 06 · 章节验收矩阵

## 使用方法

每章学完后，不用“读过了”打勾，而是留下可复查产出。通用评分是：

- 0：无产出，或只复述名词。
- 1：能解释概念，但无边界/反例/工程决策。
- 2：能从记忆重建图/表/计算，给出反例并映射到贯穿案例。
- 3：能处理未见变体，写出可测不变量、失败证据和 enforcement point。

每行至少达到 2；标记“安全关键”的行必须达到 3。若产出依赖外部实系统，前置阶段用 mock 或纸面时序代替；不因实现重而跳过理论验收。

## 毕业证据包最小清单

1. M0 + 非 Agent baseline + 30–50 个版本化 cases。
2. 统计/评测计划：graders、trials、区间、MDE/power、停止与发布门禁。
3. TypeScript/Node runtime 图、流 event reducer、Agent 状态机和不变量。
4. Context Manifest、ACL-scoped retrieval、provenance 和 Memory 删除策略。
5. Query/Command 工具契约、授权矩阵、approval proposal、sink 安全和 ACK-loss 时序。
6. 威胁模型、攻击链、纵深防御矩阵、UX 状态映射和 incident playbook。
7. 背压/预算上界、Trace/SLO/成本设计。
8. 闭卷答卷、评分表和未清零错误模型列表。
