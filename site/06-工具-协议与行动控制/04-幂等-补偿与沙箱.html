#04 · 幂等、补偿与沙箱

通过授权和审批并不意味着一次动作会干净地成功或失败。取消订单可能已在服务端提交，回执却在网络中丢失；Runtime 随后的重试、用户取消或工作进程（Worker）恢复，都可能把同一业务意图执行第二次。

本章从这种“效果已经发生，但调用者无法确认”的工程矛盾出发，把幂等、去重、补偿和受限执行连接起来。它们不是工具描述里的建议，而是执行器（Executor）、状态存储与沙箱共同强制的运行时语义。

#学习目标

  • 理解未知完成状态和重复副作用。
  • 区分幂等、去重、事务和补偿。
  • 知道语言内存安全、容器与安全沙箱的边界。

#1. 最危险的失败序列

客户端发送 command
→ 服务端提交成功
→ 响应在网络中丢失
→ 客户端看到 timeout
→ retry
→ 第二次副作用

timeout 只说明调用者没及时收到确定结果，不说明服务端没执行。

#2. 幂等键

同一业务意图的所有 retry 使用稳定 key；新的业务意图必须使用新 key。服务端需要原子地记录 key、payload fingerprint 和结果：

  • 首次请求执行并持久化结果。
  • 相同 key + 相同 intent 返回语义等价结果。
  • 相同 key + 不同 payload 应拒绝。
  • 定义 key 的作用域、保留期与并发竞争行为。

HTTP method 的理论幂等性不能自动保证业务实现正确。

#3. 去重、事务与补偿

  • Deduplication：识别重复请求/消息。
  • Transaction：在单一一致性边界内原子提交。
  • Outbox/Inbox：协调数据库状态与消息交付。
  • Compensation：在不能回滚时执行语义相反或修复动作。

补偿不是时间倒流。例如“发更正邮件”不能真正撤回已被阅读的邮件。

#4. Preview 与 Commit

高风险动作使用两阶段产品语义：

draft/preview → policy/approval → commit(idempotency_key, expected_version)

它不是通用分布式两阶段提交，但能把用户理解的提案与最终副作用绑定。

#5. 沙箱威胁面

执行任意代码或高权限工具时至少隔离：

  • Filesystem：只读根、临时目录、路径边界。
  • Network：默认拒绝、egress allowlist、DNS 与 metadata endpoint。
  • Process/syscall：子进程、系统调用、信号和 PID 边界。
  • Credentials：不把宿主秘密注入沙箱。
  • Resources：CPU、内存、磁盘、进程数、网络和时间。
  • Lifecycle：每任务清理、镜像固定、审计和销毁。

容器通常共享宿主内核，不等于强安全边界。Rust 消除一类内存安全问题，但不会自动提供文件、网络、身份或系统调用隔离。

#6. Sink 与沙箱是两个边界

沙箱限制一个进程“最多能碰到什么”；sink-specific 校验决定“这个输入能否以这种方式进入 SQL、shell、路径、URL 或 HTML”。两者都不能被 Prompt 或 Schema 取代。例如网络沙箱已禁止内网访问，URL 执行器仍要限制 scheme/host/redirect/DNS 重绑定；二者一层失效时另一层仍能缩小影响。

#纸面微实验（30–45 分钟）

画出“command 已提交但 ACK 丢失”的时序，分别推演无幂等键、相同 key+相同 payload、相同 key+不同 payload。再为文件和 HTTP 工具列出两层独立拒绝点。

#L1 后系统实验

让 mock 写工具“提交成功后丢弃响应”，对比无幂等键和有幂等键的真实副作用次数。再尝试从沙箱读取宿主凭证和向未知域名发送，验证文件与 egress 两层独立拒绝。

#常见误区

  • retry 总能提高可靠性。
  • 幂等键由模型随机生成即可。
  • Cancellation 会撤销已提交 command。
  • Compensation 等同数据库 rollback。
  • Docker/Rust 自动解决 Agent 执行安全。

#章末检查

  1. 相同幂等键、不同 payload 应如何处理？
  2. 为什么已发送邮件只能补偿，不能真正回滚？
  3. Rust 内存安全与沙箱安全分别覆盖哪些风险？

#本章小结

可靠行动的底线不是“调用成功”，而是同一业务意图可去重、未知效果可查询收敛、不可回滚效果有补偿或人工处置，且执行能力受沙箱和 sink-specific 校验共同约束。工具链至此具备了行动控制骨架，下一部分将从Agent 威胁建模检查不可信输入如何跨这些边界组合成攻击路径。
